### 2.2 任务计划

#### 2.2.1 摸清存活IP和目标端口
先汇总目标系统、服务和组件的默认端口（ports_list1），然后考虑到会采用**非标准默认端口**，可以选取其中两三个网段进行抽样全端口扫描，识别出目标端口，再次汇总（port_list2）。最终的端口清单由ports_list1和ports_list2合并得来。
然后需要进行日程规划。可以考虑按照风险项来，也可以考虑按照网段来，要体现出进度来。实际我采用了按风险项的方式进行扫描。但在此之前，需要先摸清存活IP与端口。
对3000余个C段进行存活IP探测与端口扫描，一个是用绿盟的EZ进行扫描，同时我们也可以使用nmap。扫描的服务器有3台，将3000余个网段分别分配到3个服务器上扫描，ports.txt和targets.txt都是一项一行。
##### nmap
Linux：
```bash
# 使用 `paste -sd,` 将文件中的所有换行符替换为逗号，组合成端口字符串。
nmap -p $(paste -sd, ports.txt) -iL targets.txt --min-rate 1000
# 加-sV参数，扫描的同时探测服务版本，-n禁用DNS反解提速
nmap -sV -n -p $(paste -sd, ports.txt) -iL targets.txt --min-rate 1000
```

Windows：
```powershell
nmap -p ((Get-Content ports.txt) -join ',') -iL targets.txt --min-rate 1000
nmap -sV -n -p ((Get-Content .\ports.txt) -join ',') -iL .\targets.txt --min-rate 1000
```

输出需要通过Python脚本来处理输出为CSV，格式保留IP和端口即可。
> 如果是按网段表示进度，那就需要划分网段，我这里是按风险项进行排查，所以不划分网段。


##### masscan
masscan在扫描存活IP和开放端口比nmap快，尤其是在大规模任务下，所以这一步，也可以直接使用masscan来扫描。命令如下：
Linux
```bash
sudo masscan -p $(paste -sd, ports.txt) -iL targets.txt --rate=10000
# 输出 JSON 结果到文件，并开启 Banner 抓取
sudo masscan -p $(paste -sd, ports.txt) -iL targets.txt --rate=10000 --banners -oJ scan_result.json
# 输出txt文件，-oL 会输出 port,state,ip 的简洁格式，方便后面nmap接力
masscan -p $(paste -sd, ports.txt) --rate=10000 -iL targets.txt -oL masscan_ports.txt
```

Windows
```powershell
masscan -p ((Get-Content ports.txt) -join ',') -iL targets.txt --rate=10000 -oJ result.json

masscan -p- --rate=10000 -iL targets.txt -oG masscan_output.gnmap
```


##### EZ
使用EZ的话，就比较简单，直接点点点就行了，接下来就是等“收菜”。

#### 2.2.2 识别WEB服务和非WEB服务
这里主要是针对前面扫描到的存活IP和开放端口进行服务识别，这里就先用nmap。由于前面的扫描已经快速收敛目标，最终的host（ip:port）剩余13万条。这里会把nmap扫描的指纹结果分为两类，一类是web服务，一类是非WEB服务（ssh、ftp等）。
如果是用masscan扫描存活IP和开放端口，在保存txt文件后，可以用下面的AI给的脚本接力扫服务：
```bash
#!/bin/bash
# ============================================================
# 脚本名称: masscan_to_nmap_parallel.sh
# 功能: 将 masscan -oL 输出转换为对每个 IP 的 nmap 深度扫描
# 用法: ./masscan_to_nmap_parallel.sh <masscan_ports.txt> [并发数]
# 示例: ./masscan_to_nmap_parallel.sh masscan_result.txt 10
# ============================================================

MASSFILE="$1"
CONCURRENT="${2:-8}"   # 默认并发数 8

if [ ! -f "$MASSFILE" ]; then
    echo "Usage: $0 <masscan_-oL_output.txt> [concurrent_jobs]"
    exit 1
fi

# ---------- 聚合端口函数 ----------
# 输入: masscan -oL 格式 (每行: port,state,protocol,ip)
# 输出: 每行 "IP 端口列表" (端口用逗号分隔)
aggregate_ports() {
    awk -F',' '
        {
            ip = $4
            port = $1
            if (ports[ip]) ports[ip] = ports[ip] "," port
            else ports[ip] = port
        }
        END {
            for (ip in ports) {
                print ip, ports[ip]
            }
        }
    ' "$MASSFILE"
}

# ---------- 导出函数（xargs 需要） ----------
export -f aggregate_ports   # 如果使用 bash 函数，但这里我们用管道直接执行

# ---------- 并行执行 ----------
aggregate_ports | xargs -P "$CONCURRENT" -I {} sh -c '
    set -- {}
    ip=$1
    ports=$2
    if [ -n "$ports" ]; then
        echo "[+] Scanning $ip for ports: $ports"
        # 可选：加入 -n 禁用 DNS 反向解析，-T4 提速
        nmap -sV -sC -T4 -n -p "$ports" "$ip" -oA "scan_$ip"
    fi
' _

# 等待所有后台任务完成（xargs -P 会自动等待）
echo "[✓] All scans completed. Results saved as scan_<IP>.{nmap,gnmap,xml}"
```

运行（假设 masscan 输出文件为 `masscan_ports.txt`）：    
```bash
./masscan_to_nmap_parallel.sh masscan_ports.txt 10
```

第二个参数 `10` 指定并行扫描的 IP 数量，默认 8。
**输入文件格式要求**：
必须是 `masscan` 的 **`-oL`** 格式（每行 `端口,状态,协议,IP`），例如：
```
80,open,tcp,192.168.1.1
443,open,tcp,192.168.1.1
22,open,tcp,192.168.1.2
```   
该格式可通过 `masscan -oL -p...` 直接生成。

**结果保存**：每个 IP 生成三个文件（`scan_<IP>.nmap`、`.gnmap`、`.xml`），方便后续分析。


接着通过nmap区分WEB服务和非WEB服务。这个需要根据对masscan的扫描结果进行提取，然后交给nmap识别服务，从识别结果中用正则提取出web服务（指纹是http）的host(ip:port)，剩下的就进行针对性的弱口令、默认凭证和未授权访问漏洞扫描。

**zookeeper**
真正判定"是否未授权"的是 **`getAcl /`看根节点是不是 `world:anyone:cdrwa`全开**，而不是单纯 `telnet 2181`能连，以及允许部分四字命令的执行。

```bash
# 可以未授权执行四字命令envi
nuclei -l target.txt -t network/exposures/exposed-zookeeper.yaml

```

然后看节点权限，需要先安装zkCli.sh，然后调用这个模板：

```yaml
id: zookeeper-root-acl

info:
  name: ZooKeeper Root ACL Check
  author: self
  severity: medium
  description: Check root node ACL.

self-contained: true

code:
  - engine:
      - bash

    source: |
      zkCli.sh -server {{Host}}:{{Port}} getAcl / 2>/dev/null

    matchers-condition: and

    matchers:

      - type: regex
        part: stdout
        regex:
          - "world.*anyone"

      - type: regex
        part: stdout
        regex:
          - "cdrwa"

    extractors:

      - type: regex
        part: stdout
        regex:
          - "world.*"
```

> zkCli.sh 是 ZooKeeper 官方自带的命令行客户端，作用相当于 MySQL 的 mysql、Redis 的 redis-cli。它包含在 Apache ZooKeeper 的安装包中。

也可以直接用python脚本实现扫描：
安装库：
```bash
pip install kazoo
```
脚本：
zk_acl.py
```python
from kazoo.client import KazooClient
from kazoo.security import Permissions

def detect_zookeeper_root_acl(host, port=2181):
    result = {
        "vulnerable": False,
        "acls": []
    }

    zk = KazooClient(hosts=f"{host}:{port}", timeout=5)

    try:
        zk.start(timeout=5)

        acls, _ = zk.get_acls("/")

        for acl in acls:
            item = {
                "scheme": acl.id.scheme,
                "id": acl.id.id,
                "perms": acl.perms,
                "full_access": (
                    acl.id.scheme == "world"
                    and acl.id.id == "anyone"
                    and acl.perms == Permissions.ALL
                )
            }

            result["acls"].append(item)

            if item["full_access"]:
                result["vulnerable"] = True

    finally:
        zk.stop()
        zk.close()

    return result


if __name__ == "__main__":
```

批量目标扫描，batch_zk_acl.py
```python
import csv
import socket
from concurrent.futures import ThreadPoolExecutor, as_completed

from kazoo.client import KazooClient
from kazoo.security import Permissions

TIMEOUT = 5
THREADS = 50


def port_open(host, port):
    try:
        s = socket.create_connection((host, port), timeout=TIMEOUT)
        s.close()
        return True
    except Exception:
        return False


def scan(host, port=2181):

    result = {
        "host": host,
        "port": port,
        "status": "",
        "vulnerable": False,
        "detail": ""
    }

    if not port_open(host, port):
        result["status"] = "closed"
        return result

    zk = KazooClient(
        hosts=f"{host}:{port}",
        timeout=TIMEOUT,
    )

    try:

        zk.start(timeout=TIMEOUT)

        acls, _ = zk.get_acls("/")

        result["status"] = "open"

        for acl in acls:

            if (
                acl.id.scheme == "world"
                and acl.id.id == "anyone"
                and acl.perms == Permissions.ALL
            ):
                result["vulnerable"] = True

            result["detail"] += (
                f"{acl.id.scheme}:{acl.id.id}:{acl.perms};"
            )

    except Exception as e:

        result["status"] = "error"
        result["detail"] = str(e)

    finally:

        try:
            zk.stop()
            zk.close()
        except Exception:
            pass

    return result


def load_targets(filename):

    targets = []

    with open(filename) as f:

        for line in f:

            line = line.strip()

            if not line:
                continue

            if ":" in line:
                host, port = line.split(":")
                targets.append((host, int(port)))
            else:
                targets.append((line, 2181))

    return targets


def main():

    targets = load_targets("targets.txt")

    results = []

    with ThreadPoolExecutor(max_workers=THREADS) as pool:

        futures = {
            pool.submit(scan, h, p): (h, p)
            for h, p in targets
        }

        for future in as_completed(futures):

            r = future.result()

            print(
                f"{r['host']}:{r['port']} "
                f"{r['status']} "
                f"VUL={r['vulnerable']}"
            )

            results.append(r)

    with open(
        "result.csv",
        "w",
        newline="",
        encoding="utf-8",
    ) as f:

        writer = csv.DictWriter(
            f,
            fieldnames=[
                "host",
                "port",
                "status",
                "vulnerable",
                "detail",
            ],
        )

        writer.writeheader()

        writer.writerows(results)


if __name__ == "__main__":
    main()
```

## 3. 小结

安全风险排查不能还是以攻击者的视角去进行排查，只想着找到一条路径打穿，而是要考虑扫描哪些漏洞，比如这次的认证、凭证风险和未授权访问风险专项，同时作为一项排查工作，需要每日汇报进度，那么项目进度显示就需要有据可依，比如按风险项来表示，如：
第一天完成了大规模端口推排与指纹收敛，存活资产按照风险项进行分类；
第二天开始依次并行地对风险项进行排查扫描，整理数据；
最后一天就是汇总成果，分析薄弱点，并给出治理方案。
每日汇报时汇报的内容应包括，总共规划n个风险项排查，今日已完成XXX专项扫描，整体进度如表。然后表格的内容至少可以基于上面`2.1 设置扫描目标`这一节的表，后面加上状态。如：

| 风险项    | 风险类别    | 资产数量     | 成果数量    | 状态  |
| ------ | ------- | -------- | ------- | --- |
| SSH弱口令 | 认证、凭证风险 | 6万个SSH服务 | 100个弱口令 | 已完成 |
类似的在第一天的`大规模端口推排与指纹收敛`工作如果不能再第一天完成，也应该体现出进度，比如可以分网段来扫描，并以此表现进度。不同的任务内容，应该完成合适的待办裁切，及早划分子任务，比如说网段、按部门、按漏洞类型等等。
最后就是本次排查，第一次尝试使用了AI Agent，模型是GLM5.1，64C64G的配置下，并发表现不佳，每次只能开启三个会话，但是对一些内网应用，没有现成的接口和框架知识时，Agent进行智能的自动化渗透，Agent的灵活性和通用性还是能有所体现，这次Agent的表现主要是在总控台上去发挥，还是能自己去找到登录接口并完成爆破，最终获取权限，有些问题就是Agent的范围控制并不太好，比如由于其性能有限，所以限定它一个IP和端口作为目标，它会打到别的端口上，但最终也确实发现了漏洞（623）。我没有注意到这个端口是有问题的，这属于我的知识缺陷，但是这种不按指令去打的越界，还是需要设置拦截。不过，我对于AI Agent日后自动化完成类型的排查工作依然持乐观态度，上面的计划和执行完全可以编排成自动化流水线去完成。
